Command Line Prompts for Git:

Directories & Navigation:

  • Create a Directory: mkdir [Directory Name]
  • Navigate to Directory "Change Directory": cd [Directory Name]/
  • Back Out of Current Location: cd ..
  • Navigate to Absolute Path: cd /
  • Navigate to Top Level Directory: cd mnt
  • List Files in Directory: ls
  • Remove Directories (Delete): rm -rf [Directory Name]

Files:

  • Create a File: touch [filename.extension]
  • Delete a File: rm [filename.extension]

Git:

  • Check Git Version: git --version
  • Clone a Repository: git clone [repository html from github (starts with "http://")] [name you'd like to give the repository - use _ for spaces]
  • Navigate to the Repository: cd [repository name from above]
  • Get Status of Git: git status
  • Add file to Staging Area: git add [file name and extension (i.e. README.md)]
  • Remove file from Staging Area:
  • Commit file to Local Repository: git commit -m "[message for git commit]"
  • Configure Git User Email: git config user.email "[user email address]"
  • Configure Git User Name: git config user.name "[user name]"
  • Push the Commits to Remote Repository(GitHub): git push origin [origin name (i.e. main, master, etc.)]
  • See the Changes to a File: git diff [filename.extension]
  • Get Log of Commits: git log
  • Undo Changes to File git checkout [filename.extension]
  • To Ignore File Changes (If you had Auto-generated directories, etc.):
    1. Create a GitIgnore File: touch .gitignore
    2. Add file names & extensions to the above .gitignore file
    3. Add & Commit All Files
    4. Remove All Cached Files git rm -r -- cached .
    5. Add & Commit All Files

Terminology:

  • Repositories: Store the full history and source control of a project.
  • Directories: Projects in their CURRENT state in time. (Whereas repositories contain all historical versions)
  • Working Directory: Your project at its current state in time.
  • Staging Area: Bundle of all the modifications to the project that are going to be committed.
  • Commit: Similar to taking a snapshot of the current state of the project, then storing it on a timeline. Committing saves the git to your local repository.
  • Push: This pushes the git to the remote repository.
